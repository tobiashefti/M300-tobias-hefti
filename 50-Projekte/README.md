# Handlungsziele
1. Erklärt den Zweck des gewählten Service in ein paar Sätzen. 
2. Beschreibt den Aufbau / die logische Struktur eures Projektes.
3. Konfigurieren Sie den Dienst und setzen Sie eine Monitoring Lösung ein. 
   [Beispiel für Docker](https://chatgpt.com/share/698afccb-daec-800d-8dd8-be803d99ff6e)
4. Konfigurieren Sie die Netzwerkverbindung. Welche Ports benötigt Ihr Dienst?
5. Zeigen Sie, wie Ihr Hostsystem mit Containern interagiert. (Beispiel Volumes)
6. Dokumentieren Sie Fehler, wenn sie auftreten. Zeigen Sie, wie Sie die Fehler 
   behoben haben.
7. Ergänzen Sie die Dokumentation noch wo nötig. Es braucht keine langen 
   Fliestexte, aber 1-3 Sätze geben viel Kontext für eine Person, die Ihr 
   Projekt noch nie gesehen hat.
